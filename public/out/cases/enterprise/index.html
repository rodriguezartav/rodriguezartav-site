What's in Sixth?

A framework for creating robust, maintainable business macro-applications hosted entirely on the cloud with affordable development and operating costs.

  • Salesforce.com (Force.com)
    Database , Security and Administrative Interfaces like Reporting and Listings. A business designed & programable platform used for creating process , workflows and react to changes in data.
  • Sinatra / Rails
    Standard based and robust Integration Hub; is the center of the system that allows mobile, tablet, offline and web applications to send and receive data with force.com.
  • Node-Js
    The latest invention in software, a powerful server that is able to handle millions of simultaneous requests from intranet and customer facings apps. Is the power-horse behind beautiful and engaging web and mobile applications.
  • Foundation
    A design framework for creating slick designs, with a hidden power. It helps build adaptable designs for desktop, tablets and mobiles phones; Increases production of application in ~400%.
  • SpineJs
    Makes applications responsive, entertaining and fast. A front-end (client side) framework for creating robust and maintainable apps in a fraction of the time.
  • Sixth
    The glue that joins them all. Sixth is a set of patterns and procedures that makes each of the parts of a modern cloud hosted enterprise system perform as with the accuracy of the Space Shuttle. Standards based, 100% Open Source.

Cloud Hosting

This term is actually a contradiction, since in the Cloud there is no hosting. You purchase "pipeline" instead of space, pay for what you use.

The main system and database is a service provided by Salesforce.com, which also includes the Administrative Interface like listings, reports, dashboards , etc

Each web and mobile application along with API Integration Hubs is hosted on Heroku, a Salesforce.com company that revolutionized the Open Source Community by providing an extremely efficient , scalable and affordable platform to deploy applications in seconds.

The code , organized , tested and object oriented is stored in separated repositories; Also in the cloud allowing distributed teams to work on the same projects with out conflicts with 100% reliability.
