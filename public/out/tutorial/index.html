Static Blog in NodeJS

Background

First we used to build html web sites with Adobe's Dreamweaver, using templates to be follow DRY principles. Next we moved toward CMS like Drupal , Wordpress.

We got accostumbed to working with dynamic database driven sites, however most of the time those sites remain static for a long period of time. The reason we kept using such patters for blogs and sites was because of how complicated it used to be to edit html, ftp changes , etc.

Trend

With Rails and Sinatra many of us started using templates and libraries like Jekill to generate database-free static web sites that we could quickly create and update in our favorite template engine/language.

Still Rails seemed like an overkill for my personal web site, consisting of a couple of pages, I also wanted to try and build a simple static blog with NodeJS

Docpad

I was thinking that it was far better to compile the html for my static site locally, rather than using a templating engine like Jade. Even thou it was important to avoid copy&pasting my headers and footers all over the place; It was worst ( for me ) to start tweeking my NodeJS server that I got set up for my Async UI's.

So I started the usual Google research dance , looking specifically for a Jekyll version for NodeJS. Found Docpad, which was exactly what I was looking for and it turned out to have pretty powerful features that I didn't foresee

As an example, Docpad has a way to do in-context Editing, but in may case a way to read meta data from files in folder was much more interesting so that I could keep my blog posts separated and list them "dynamically".

Getting Started

The first thing is to head to Doc Pad @ Github and install it using NPM.

I think the instructions are very clear , the installation and services are almost stable. ( sometimes take too long to finish ) So that anybody can get started right away

In order to use my prefered enviroment you should install Spine App and Hem. npm install -g Spine.App Hem

A better environment

We all have our own preferences, I work plenty with Async UI's and start all node projects with Spine App. This also sets up Stylus and such. I also migrated Foundation Framework to Stylus and made it fit perfectly with Spine.

Spine got Hem a development server that takes care of Stylus and Coffeescript, and Docpad has its own server however it doesn't have Common JS Modules or the css concatenation of a power-horse like Spine and Hem.

So I use both, inside of Spine public Directory ( NodeJS Express Static Server ) I created the basic Docpad app , then created a terminal script to Compile the Docpad part and copied from the Docpad Output directoru (/public/out) to the public directory(/public) which takes care of all the URL usage in Docpad

Step by Step

Create new Spine App: spine app blog which will create a folder named blog and inside it another one named public. The rest is out of scope.

Next go ahead scaffolding Docpad simplest App into the /public folder. git clone git://github.com/balupton/canvas.docpad.git

Inside the canvas.docpad folder , copy the /src and /out folders into the /public folder.Erase /public/canvas.docpad

Docpad compiles the content of /src into /out. I am sure there are far better ways of making docpad deploy to the /public folder or using Guard or Watch to move /public/out/* into /public ; however for me the simplest and quicker way turn out to be a script

In the project root type hem server, and every time you make a change to anything inside the public/src run the script. Docpad also comes with an automatic compiler, but is far to slow for my taste; I run faster by cmd-tab and two terminal windows.

Responsive Design with Foundation

Motivation

We've all worked with Grid 960; at least thats how I got introduced into css frameworks. I am one that liked the idea since the get go and have evolved with it.

On the way I discovered Sass and liked the indentation and variables, then moved to compass so I could easily write css3. When started playing with NodeJS I moved to LESS; and these days Stylus is king of css.

CSS is great and is , in itself, a great framework all together. Normally us designers have our own base style sheet and reusable libraries; I find my self a good designer and know css like the palm of my hand, however I am sure there are far better designers than myself out there.

The search for better css code and more importantly a solution to designing for different screen sizes made me understand that "Responsive Design was the way to go"

Zurb Foundation

Zurb is one of those shops that excel at what they do, they are one of the best design agencies in the world; So their work is interesting to follow, especially when they create a framework for responsive design and Open Source it.

Introducing Zurb Foundation, an amazing css3/html5 framework for creating responsive designs that adapt to screen size by showing, hiding and modifying elements css according to current screen size.

Zurb Foundation also feature what other frameworks does, like buttons, alerts, tabs, grid and such. Its beautiful, but what I like the most is the intelligent and simple way they cater for mobil design; making it effortless to create an app that looks perfect on a 1200px screen and on a 320px as well.

How to use it - Part I

As with most open source frameworks, authors try to make it easy for new comers to get started with their library. Foundation is not the exception; You can just download a base site. However sometimes the real life scenarios where we deploy such libraries is different from what authors imagined.

I hardly us Rails for Front-End components, however Rails-Foundation makes it as simple as it can be to integrate Foundation into a Rails App. For me, this is not fully beneficial yet.

The problem with doing things custom for a project is that you can not take advantage of the auto-pilots in most modern frameworks like rails; However it does give you the control and separation of concerns necessary to create innovative and maintainable projects.

How to use it - Part II

First I ported Zurb Foundation into Stylus and made sure all of its libraries work properly; I had to get rid of a IE Gradient Filter since it was not compiling.

Then I managed to integrate Zurb Foundation into SpineJS for creating Async UI's , which also works as well with Docpad which is how this site was created in less than 15 hours start to finish.

All you have to do is drop the Foundation port to stylus into the css folder and import the required Foundation Plugins in the index.styl file. Then you can start creating amazing layouts with such simplicity that you will be amazed at what happends when you shrink the browser screen or preview on a tablet.

Tips

The best way to learn is by looking at Zurbs work, especially the marketing site, start by cloning this work which is why they share it on their repository.

Plan you design ahead, when working with Zurb Foundation is a little different than Sass; Not as many mixings necessary, since layout is controller by markup. Example class="columns six" will take care of position and sizing.

Follow this sheet sheet of classes so that its easier to code using your favorite templating engine.
